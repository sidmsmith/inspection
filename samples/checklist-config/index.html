Checklist Configuration — UI Samples

PO checklist only · Manhattan theme · local mockups (no GitHub deploy)

Authenticate with any ORG (e.g. SS-DEMO) on each sample. Data is loaded from the current PO checklist in config/checklists.json. Signature, damage diagram, and photos are fixed — only questions are editable in these mocks.

A — Split + live preview

Question list on the left, inspection preview on the right. Click a row to edit inline. Best for seeing changes immediately.

B — Expandable cards

Each question is a card you expand to edit. Option chips for dropdowns. Sticky preview column on desktop.

C — Nav + detail

Flowthrough-style line navigator on the left, full editor on the right. Good when you have many questions.

How to run locally

cd inspection
npm start
# open http://localhost:3000/samples/checklist-config/

Or open the HTML files directly — use a local server for fetch-free scripts (recommended).
